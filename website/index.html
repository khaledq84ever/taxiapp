FEATURES

Everything you need
in one platform

Designed for passengers, drivers, and platform administrators.

⚡

Real-Time GPS Tracking

Socket.io WebSocket connection updates driver position every 5 seconds on Google Maps. Zero lag.

🔐

OTP Authentication

Phone-based login via Twilio SMS. JWT tokens with role-based access: Passenger, Driver, Admin.

💰

Smart Fare Engine

Haversine distance formula. 5 SAR base + 2.5 SAR/km. 20% platform fee auto-deducted from driver earnings.

💳

Secure Payments

Stripe payment intents for card payments. Cash option always available. SAR currency. Mada-ready.

🔔

Push Notifications

Firebase FCM for instant alerts: trip accepted, driver arrived, ride completed, payment received.

📊

Admin Dashboard

Approve or reject driver applications. Monitor all live trips. View revenue stats. Manage users.

⭐

Two-Way Ratings

Passengers rate drivers. Drivers rate passengers. 1–5 star scores auto-update driver average rating.

🇸🇦

Saudi Market Ready

SAR currency, +966 phone format, Arabic RTL support, Riyadh/Jeddah/Dammam city coverage.

🚀

Production Deployed

Live on Railway with PostgreSQL database. GitHub CI/CD auto-deploys on every push. Fully scalable.

REST API

Developer-First API

All endpoints return JSON. Authenticate with Bearer JWT token.

Send OTP Estimate Fare Book Ride
POST https://taxiapp-api-production.up.railway.app/api/v1/auth/send-otp 
// Request body
{
  "phone": "+966501234567"
}

// Response (development mode shows the code)
{
  "message": "OTP sent",
  "code": "482910"
}

// Then verify the OTP
POST /auth/verify-otp
{
  "phone": "+966501234567",
  "code":  "482910"
}

// Returns JWT token + user object
{
  "accessToken": "eyJhbGciOiJIUzI1NiJ9...",
  "user": {
    "id":    "clxyz123...",
    "phone": "+966501234567",
    "role":  "PASSENGER"
  }
}
