================================================================
MASTER BUILD PROMPT
================================================================

Build a full-stack taxi/ride-sharing app called "TaxiApp" similar to Uber and Careem,
targeting the Saudi Arabian market.

The app has 3 sides:
1. Passenger mobile app (React Native)
2. Driver mobile app (React Native - same codebase, different mode)
3. Admin web dashboard (React.js)

Backend: Node.js + NestJS + PostgreSQL + Socket.io + Redis

Core features:
- Phone number OTP registration (Twilio)
- Google Maps with real-time driver tracking
- Ride booking with fare estimation
- In-app payments (Stripe + cash option)
- Driver document verification flow
- Push notifications (Firebase FCM)
- Arabic + English language support (RTL)
- Saudi city coverage (Riyadh, Jeddah, Dammam)

Build production-ready, scalable, and secure code.
Use JWT auth, role-based access (passenger/driver/admin).
Follow REST API standards with proper error handling.

- id, passenger_id, driver_id
- pickup_address, pickup_lat, pickup_lng

- status (requested/accepted/arrived/started/completed/cancelled)
- fare_estimate, final_fare
- payment_method (cash/card)
- payment_status
- created_at, started_at, completed_at

TABLE: Ratings
- id, trip_id, rater_id, rated_id
- score (1-5), comment
- created_at

TABLE: Payments
- id, trip_id, amount, currency
- method, status, stripe_payment_id
- created_at

TABLE: Notifications
- id, user_id, title, body
- type, is_read, data (JSON)
- created_at

PHASE 3 - BACKEND API ENDPOINTS (Week 2-3)
================================================================

AUTH
  POST /auth/send-otp           Send SMS OTP to phone
  POST /auth/verify-otp         Verify OTP + return JWT token
  POST /auth/refresh-token      Refresh JWT token

USERS
  GET  /users/profile           Get my profile
  PUT  /users/profile           Update profile
  POST /users/upload-photo      Upload profile picture

DRIVERS
  POST /drivers/register        Submit driver application
  GET  /drivers/status          Check approval status
  PUT  /drivers/toggle-online   Go online/offline
  PUT  /drivers/location        Update GPS location
  GET  /drivers/earnings        View earnings history
  GET  /drivers/nearby          Get nearby available drivers

TRIPS
  POST /trips/estimate          Calculate fare estimate
  POST /trips/request           Passenger requests a ride
  GET  /trips/:id               Get trip details
  PUT  /trips/:id/accept        Driver accepts trip
  PUT  /trips/:id/arrived       Driver arrived at pickup
  PUT  /trips/:id/start         Trip started
  PUT  /trips/:id/complete      Trip completed
  PUT  /trips/:id/cancel        Cancel trip
  GET  /trips/history           My trip history

PAYMENTS
  POST /payments/create-intent  Create Stripe payment intent
  POST /payments/confirm        Confirm payment

RATINGS
  POST /ratings                 Submit rating after trip

ADMIN
  GET  /admin/drivers/pending   List pending driver applications
  PUT  /admin/drivers/:id/approve
  PUT  /admin/drivers/:id/reject
  GET  /admin/users             All users list
  GET  /admin/trips             All trips list
  GET  /admin/stats             Dashboard statistics


================================================================
PHASE 4 - REAL-TIME SOCKET.IO EVENTS (Week 3)
================================================================

Driver Emits:
  driver:location-update    { lat, lng, driverId }
  driver:trip-accepted      { tripId, driverInfo, eta }
  driver:arrived            { tripId }
  driver:trip-started       { tripId }
  driver:trip-completed     { tripId, finalFare }

Passenger Emits:
  passenger:trip-request    { pickup, dropoff, passengerId }
  passenger:cancel-trip     { tripId }

Server Emits to Passenger:
  server:driver-found       { driverInfo, eta, location }
  server:driver-location    { lat, lng }  (live updates)
  server:no-drivers         {}
  server:trip-update        { status }

Server Emits to Driver:
  server:new-trip-request   { tripDetails, passengerInfo }
  server:trip-cancelled     { tripId }
